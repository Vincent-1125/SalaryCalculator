# 年总包和到手薪资计算器 — 说明文档

> 本文档基于 `SalaryCalculator.xlsx` 整理，总结该薪资计算器的结构、公式逻辑与配置信息。

---

## 一、整体结构

Excel 文件包含两个工作表：

| 工作表 | 说明 |
|--------|------|
| **Annual salary calculator** | 主计算表，包含薪资计算、变量配置、每月个税明细 |
| **Info** | 税率表、各城市五险一金基数上下限、专项附加扣除查找表 |

主计算表分为三大区域：

| 区域 | 列范围 | 功能 |
|------|--------|------|
| 年总包和到手薪资计算 | B–D | 计算年总包、各项扣款、税后到手金额 |
| 变量配置 | F–H | 可调节的参数（月薪、年终、专项扣除等） |
| 每月缴纳个税金额计算 | J–P | 按累计预扣法计算 1–12 月每月个税与到手 |

## 三、年总包计算逻辑（B–D 列）

### 3.1 收入项

| 项目 | 公式 | 说明 |
|------|------|------|
| **工资** | `=G4×G6+G5` | 月薪 × 12 + 年津贴 |
| **年终** | `=G4×G7` | 月薪 × 年终期数（默认 4） |
| **总包** | `=C4+C5` | 工资 + 年终 |

### 3.2 社保公积金（五险一金）

所有社保扣款以 **实际缴费基数** 为准，基数为 `MEDIAN(月薪, 下限, 上限)`，即限制在上下限之间。

| 项目 | 公式 | 说明 |
|------|------|------|
| **五险合计** | `=MEDIAN(G4, G25, G24) × 10.5% × 12` | 基数 × 费率 × 12 月 |
| **养老保险** | `=MEDIAN(G4, G25, G24) × 8% × 12` | |
| **医疗保险** | `=MEDIAN(G4, G25, G24) × 2% × 12` | |
| **失业保险** | `=MEDIAN(G4, G25, G24) × 0.5% × 12` | |
| **公积金** | `=MEDIAN(G4, G27, G26) × 12% × 12` | 公积金有单独的基数上下限 |

### 3.3 应纳税所得额与应纳税额

| 项目 | 公式 | 说明 |
|------|------|------|
| **应纳税所得额（工资）** | `=MAX(0, C4 - G28 - G8 - C12 - C16)` | 工资收入 - 免征额 - 专项扣除 - 五险 - 公积金 |
| **应纳税所得额（年终）** | `=MAX(0, C5)` | 年终全额作为应税所得 |
| **应纳税额（工资）** | 按 Info 工资税率表累进计算 | `应税所得额 × 税率 - 速算扣除数` |
| **应纳税额（年终）** | 按 Info 年终奖税率表计算 | `年终 × 税率 - 速算扣除数` |

### 3.4 到手金额

```
到手（含公积金）= 总包 - 五险 - 个税(工资) - 个税(年终) + 公积金
```

即：`C18 = C6 - C12 - C10 - C11 + C16`

> 注：公积金计入到手，因为个人缴存部分仍归属个人。

### 3.5 总计扣款

```
C22 = C10 + C11 + C13 + C15 - C14 - C16
```

## 四、每月个税计算（J–P 列）

采用 **累计预扣法**，逐月计算个税：

| 列 | 名称 | 公式逻辑 |
|----|------|----------|
| J | 月份 | 1–12 |
| K | 当月工资 | 固定为 `$G$4`（月薪） |
| L | 累计应纳税所得额 | `=SUM($K$4:$K当前行) - P4 × J当前行` |
| M | 累计应纳税额 | `=MAX(0, L) × 税率 - 速算扣除数`（查 Info 工资税率表） |
| N | 本月纳税额 | `=M当月 - M上月` |
| O | 本月到手 | `=K - N - (五险 + 公积金) / 12` |
| P | 每月扣除额 | `=5000 + G20 + (C12 + C16)/12`（起征点 + 专项扣除 + 月均社保公积金） |

## 六、使用说明

1. **修改月薪**：修改 `G4`（月薪），所有计算自动更新
2. **修改年终期数**：修改 `G7`，调整年终奖月数
3. **修改津贴**：修改 `G5`，调整年度津贴
4. **选择专项扣除**：修改 `G13:G18` 的下拉选项，金额自动从 Info 表查询
5. **切换城市**：修改 `G24:G27` 的社保/公积金基数上下限为对应城市的值
6. **修改公积金比例**：修改 `G9`

## 八、关键设计要点

1. **社保基数封顶保底**：使用 `MEDIAN(月薪, 下限, 上限)` 确保缴费基数在法规范围内
2. **累计预扣法**：每月个税采用累计预扣，与实际工资薪金个税计算一致
3. **年终单独计税**：年终奖使用独立的税率表（Info A11:C18），速算扣除数不同
4. **公积金计入到手**：公积金个人账户余额归属个人，故算入到手金额
5. **专项扣除灵活配置**：通过下拉选项 + VLOOKUP 实现六类专项附加扣除的动态配置
